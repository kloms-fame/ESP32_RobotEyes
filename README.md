# RobotEyes 桌面陪伴机器人
> 基于 ESP32-C3 的双 OLED 拟人化表情桌面伴侣 | 舵机眉毛 | 摇杆视线控制 | 生命感微动作

---

## 项目简介
RobotEyes 是一个极具灵性与工程美感的桌面硬件项目，以 ESP32-C3 为主控，通过两块 128×64 单色 OLED 模拟机器人双眼，搭配两个 SG92R 舵机作为动态眉毛，结合摇杆与 ADC 键盘实现多模态交互。

项目摒弃了生硬的像素跳转与机械运动，引入**平滑缓动算法**与**微动作生命引擎**，让机器人拥有自然的眨眼、视线漂移、眉眼联动效果，仿佛拥有生命一般俏皮可爱。系统采用事件驱动架构与 FreeRTOS 多任务设计，模块化程度高，可扩展性强，既是桌面治愈系摆件，也是嵌入式动画与交互开发的优秀实践案例。

| -------- | ------------------------------------ | ---- | --------------------------------------- |
| 主控     | ESP32-C3 开发板                      | 1    | 系统控制、WiFi 扩展、FreeRTOS 调度      |
| 左眼屏幕 | SSD1306 0.96寸 白色 OLED（四针 I2C） | 1    | 左眼图像渲染，128×64 分辨率             |
| 右眼屏幕 | GME12864 白色 OLED                   | 1    | 右眼图像渲染，128×64 分辨率             |
| 执行器   | SG92R 微型舵机                       | 2    | 分别控制左眉、右眉角度，模拟表情        |
| 输入 1   | LY-ADCKkeyboard-01 ADC 分压键盘      | 1    | 8 按键，单 ADC 引脚读取，切换表情与模式 |
| 输入 2   | XY 双轴模拟摇杆（带按键）            | 1    | 控制视线朝向，支持按键触发交互          |

### ESP32-C3 引脚分配
> 全部使用 ADC1 通道，避免 WiFi 开启时 ADC2 不可用的问题
| GPIO   | 功能           | 类型     | 备注                        |
| ------ | -------------- | -------- | --------------------------- |
| GPIO 0 | 摇杆 X 轴      | ADC1_0   | 控制视线水平方向            |
| GPIO 1 | 摇杆 Y 轴      | ADC1_1   | 控制视线垂直方向            |
| GPIO 2 | ADC 键盘信号   | ADC1_2   | LY-ADCKkeyboard-01 电压检测 |
| GPIO 3 | 摇杆按键 (SW)  | 数字输入 | 全局确认/安全复位键         |
| GPIO 4 | 左眉毛舵机 PWM | PWM 输出 | SG92R 舵机 1                |
| GPIO 5 | 右眉毛舵机 PWM | PWM 输出 | SG92R 舵机 2                |
| GPIO 8 | I2C SDA        | 通信     | 双 OLED 共用数据总线        |
| GPIO 9 | I2C SCL        | 通信     | 双 OLED 共用时钟总线        |

> 双 OLED I2C 地址说明：SSD1306 默认地址 `0x3C`，GME12864 通过硬件修改地址为 `0x3D`，实现同一条 I2C 总线上的独立刷新与异步显示。

## 核心功能特性
### 1. 双 OLED 独立眼动系统
- **双眼异步渲染**：两块屏幕拥有独立帧缓存与独立渲染线程，支持完全不同步的动作
  - 可实现单眼眨眼、左右眼视线方向不一致、交替扫视等拟人化效果
  - 也支持联动播放完整表情动画，动作同步无延迟
- **分层视线控制**：摇杆不直接控制像素坐标，而是输入**注视目标 (Look Target)**
  - 内部通过平滑插值算法计算眼球实际位置，运动带有物理惯性，无瞬移感
  - 支持上下左右全方向注视，眼球移动带缓动回弹效果
- **12+ 预设表情库**：内置中性、开心、难过、愤怒、惊讶、困倦、害羞、好奇、兴奋、爱心眼、星星眼、眩晕等全套表情
  - 每个表情封装为独立结构体，包含左右眼位图、动画帧数、播放速度、对应眉毛角度

### 2. 舵机拟人眉毛系统
- 两个 SG92R 舵机分别对应左眉、右眉，通过 50Hz 硬件 PWM 精准控制角度
- 表情联动：切换表情时眉毛自动匹配对应姿态
  - 愤怒：双眉下压内八
  - 惊讶：双眉瞬间高抬
  - 难过：八字眉下垂
  - 开心：双眉微微上扬
- 视线联动：眼球看向侧边时，同侧眉毛跟随小幅抬高，动作自然协调
- 支持自定义角度微调与校准，角度参数可持久化保存

### 3. 生命感微动作引擎 (Micro Animation)
系统内置 Idle 态随机微动作，让机器人在无人操作时也仿佛拥有呼吸感：
- **非同步随机眨眼**：双眼眨眼存在 20-50ms 时间差，偶尔出现半眨眼状态，间隔 2-6 秒随机变化
- **视线微漂移**：摇杆居中时，眼球在中心位置做微小的无规则震颤与缓慢漂移，模拟人类注视时的自然眼动
- **眉毛微动**：静止状态下眉毛以极小幅度缓慢起伏，无机械僵硬感
- **缓动过渡**：所有表情切换、视线移动、眉毛运动均采用 EaseInOut 缓动算法，无生硬跳变

### 4. 多模态输入控制
- **XY 摇杆**：全向控制视线注视方向，支持快速拨动、慢速移动、长按按键等多种交互手势
- **8 键 ADC 键盘**：
  - 6 个表情快捷键，一键切换预设表情
  - 模式切换键、功能确认键
  - 支持短按、长按、双击、长按组合键检测

### 5. 进阶智能交互
- 情绪自动触发：根据摇杆操作幅度与静止时长，自动切换对应情绪
- 眉毛微调模式：独立校准左右眉毛角度，自定义默认姿态
- 互动反馈特效：快速操作摇杆触发对应动画反馈
- 全局安全复位：任意状态下长按摇杆按键，强制回归正常模式

### 6. 低功耗休眠机制
- 闲置 30 秒无任何输入自动进入休眠
- 休眠时双屏亮度降至 10%，显示闭眼动画
- 舵机归中放平，降低功耗
- 任意按键、任意摇杆操作即可瞬间唤醒

## 完整操作指南
### 摇杆控制说明
| 操作                 | 效果                                                         |
| -------------------- | ------------------------------------------------------------ |
| 缓慢推动摇杆         | 视线平滑跟随摇杆方向移动                                     |
| 快速大幅拨动摇杆     | 自动触发「惊讶」表情，瞳孔缩小、眉毛高抬                     |
| 快速左右反复拨动摇杆 | 触发「快速扫视」特效，双眼左右来回扫视，眉毛同步小幅摆动     |
| 摇杆居中静止 10 秒   | 自动切换为「犯困」表情，眼皮半垂、眉毛放松                   |
| 短按摇杆按键         | 触发单次眨眼                                                 |
| 长按摇杆按键 (>1.5s) | 连续眨眼；同时为**全局安全复位键**，任意状态下强制回归正常模式 |

### LY-ADCKkeyboard-01 按键说明
| 按键 | 短按功能               | 长按功能                               |
| ---- | ---------------------- | -------------------------------------- |
| S1   | 眉毛微调模式：确认保存 | 长按 2 秒进入/退出眉毛独立微调模式     |
| S2   | 切换「开心」表情       | -                                      |
| S3   | 切换「愤怒」表情       | -                                      |
| S4   | 切换「难过」表情       | -                                      |
| S5   | 切换「惊讶」表情       | -                                      |
| S6   | 切换「害羞」表情       | -                                      |
| S7   | 切换「好奇」表情       | -                                      |
| S8   | 下一个表情（循环切换） | 长按退出自动表情循环模式，切回手动控制 |

### 模式切换详解
#### 正常模式（默认）
- 摇杆控制视线朝向
- 按键切换表情
- 微动作系统自动运行
- 闲置超时自动休眠

#### 眉毛微调模式
- 进入方式：长按 S1 2 秒，左屏显示 `L-Ang`，右屏显示 `R-Ang`
- 操作方式：
  - 摇杆 Y 轴：调节左眉毛角度
  - 摇杆 X 轴：调节右眉毛角度
- 保存退出：短按 S1 保存当前角度并退出；再次长按 S1 不保存退出
- 校准值自动存入 Flash，重启后生效

#### 自动循环模式
- 系统自动轮播预设表情，搭配随机微动作
- 退出方式：长按 S8 键，切回手动控制模式

#### 休眠模式
- 进入条件：30 秒无任何输入
- 表现：屏幕变暗、显示闭眼动画、眉毛归中放平
- 唤醒方式：按任意键、拨动摇杆

## 进阶交互玩法
### 情绪自动触发
无需手动按键，系统会根据你的操作自动反馈情绪：
1. **惊讶触发**：摇杆 X/Y 轴变化速率超过阈值（快速猛打摇杆）
   - 效果：瞳孔瞬间缩小、眼眶放大、双眉猛然抬高，维持 1.5 秒后缓慢回落
2. **犯困触发**：摇杆持续居中、无按键操作满 10 秒
   - 效果：眼皮缓慢下垂、眨眼频率降低、眉毛放松，进入慵懒状态
3. **退出自动循环**：长按 S8 键，终止自动表情轮播

### 眉毛独立微调模式
- 支持 0-180° 全范围精细调节
- 调节过程实时生效，所见即所得
- 校准参数持久化存储在 NVS 分区，掉电不丢失
- 支持恢复出厂默认角度

### 互动反馈特效
1. **快速扫视**：快速左右反复拨动摇杆
   - 双眼高频左右扫视，眉毛同步跟随小幅摆动，营造警觉、张望的生动效果
2. **连续眨眼**：长按摇杆按键
   - 双眼持续快速眨眼，表现出开心、期待的拟人反馈

### 休眠省电机制
- 休眠时 OLED 进入低功耗模式，亮度降至最低
- 舵机 PWM 输出优化，降低静态功耗
- 唤醒响应延迟 < 100ms，操作无卡顿感
- 休眠时长可通过配置文件自定义

## 系统架构设计
### 分层软件架构
采用四层解耦架构，高内聚低耦合，便于扩展与维护：
```
┌─────────────────────────────────────────────┐
│ 应用层：表情管理 / 行为引擎 / Web控制台      │
├─────────────────────────────────────────────┤
│ 核心服务层：事件总线 / 状态机 / 配置管理     │
├─────────────────────────────────────────────┤
│ 驱动层：OLED渲染 / 舵机控制 / 输入采集       │
├─────────────────────────────────────────────┤
│ 硬件抽象层：I2C / SPI / ADC / PWM / WiFi    │
└─────────────────────────────────────────────┘
```

### 事件驱动机制
系统基于 FreeRTOS 队列实现 EventBus 事件总线：
- 所有输入（按键、摇杆、定时器、Web命令）统一封装为事件，推入全局队列
- 渲染、舵机、行为等模块独立消费事件，模块间无直接函数调用
- 完全解耦输入与业务逻辑，新增功能无需修改原有模块

### FreeRTOS 多任务调度
| 任务名称     | 优先级 | 栈大小 | 核心职责                           |
| ------------ | ------ | ------ | ---------------------------------- |
| InputTask    | 高     | 2KB    | ADC 键盘扫描、摇杆采样、事件推送   |
| RenderTask   | 中     | 4KB    | 双 OLED 渲染、动画帧更新、视线插值 |
| ServoTask    | 中     | 2KB    | 舵机角度平滑过渡、眉毛微动作       |
| BehaviorTask | 中     | 2KB    | 随机微动作、情绪状态机、自动行为   |
| WebTask      | 低     | 4KB    | WiFi 管理、Web 控制台服务（可选）  |
