## 🔬 核心问题诊断

### 问题 1：眉毛动画未真正驱动舵机（致命 Bug）

你的 `main.cpp` 中 `EVT_JOYSTICK_MOVE` 事件处理里，每次摇杆移动都会调用 `servo_set_target()`，但**只会在摇杆事件触发时更新一次舵机**。而 `eye_expr_update()` 中每帧计算的 `brow_offset_l / brow_offset_r` 动画偏移（呼吸、震颤、抽泣、弹跳）**从未同步到舵机**。

**证据**：从串口日志看，切换到 Happy 后 1.5 秒就自动回弹了，期间眉毛舵机只在切换瞬间动一下，后续弹跳动画完全没输出到物理舵机。

### 问题 2：1.5 秒自动回弹导致循环动画无法展示

Sleepy（2.5 秒周期）、Sad（2.2 秒泪滴）、Angry（持续震颤）都需要**长按锁定**才能看到完整效果。但用户短按后 1.5 秒就回弹了。

### 问题 3：各表情的具体缺陷

| 表情          | 问题                         | 根因                                                         |
| ------------- | ---------------------------- | ------------------------------------------------------------ |
| **Happy**     | 不够开心，笑眼像 `><` 很生硬 | `draw_pupil_happy` 用直线段拼弧线，没有圆润感；上眼皮未参与形成笑弧 |
| **Angry**     | 不够凶狠                     | `lid_slope=0.9` 不够吊梢；竖缝太宽；眉毛震颤频率低；无红光闪烁 |
| **Sad**       | 不够悲伤，泪滴不明显         | 下眼皮 `lid_bottom=0.28` 浮肿感不足；泪滴太小（半径 3px）；无眼底大面积积水 |
| **Surprised** | 效果很差                     | 上下眼皮未完全撑开；电波只有 8 向且闪烁慢；眉毛弹跳幅度小    |
| **Sleepy**    | 没有反应                     | 周期 2.5s > 1.5s 回弹时间；**下眼皮完全不动**；无"惊醒"节奏  |
| **Skeptic**   | 不够完美                     | 大小眼差异不够极致；无斜视感；眉毛静态无动态                 |
| **Excited**   | 爱心很丑                     | `draw_pupil_heart` 用 `drawDisc` + `drawTriangle` 拼接，边缘锯齿严重；不对称 |

```cpp
static void draw_pupil_happy(U8G2* disp, const EyeGeom_t* g) {
    int16_t cx = g->pupil_cx, cy = g->pupil_cy, r = g->pupil_r;
    int16_t hr = r * 3 / 4;
    if (hr < 2) hr = 2;
    
    /* 上弯弧：笑眼上缘，圆润饱满（椭圆上半部分） */
    for (int16_t dy = -hr; dy <= 0; dy++) {
        float ratio = 1.0f - (float)(dy + hr) * (dy + hr) / (float)(hr * hr);
        if (ratio < 0) ratio = 0;
        int16_t dx = (int16_t)(r * 0.78f * sqrtf(ratio));
        if (dx < 1) dx = 1;
        disp->drawHLine(cx - dx, cy + dy, dx * 2 + 1);
    }
    /* 下弯弧：笑眼下缘，更宽更弯 */
    for (int16_t dy = 1; dy <= hr; dy++) {
        float ratio = 1.0f - (float)dy * dy / (float)(hr * hr);
        if (ratio < 0) ratio = 0;
        int16_t dx = (int16_t)(r * 0.78f * sqrtf(ratio));
        if (dx < 1) dx = 1;
        disp->drawHLine(cx - dx, cy + dy, dx * 2 + 1);
    }
    
    /* 眼角小星星闪烁（开心的光芒） */
    uint32_t ms = millis();
    uint8_t phase = (ms / 250) % 4;
    disp->setDrawColor(1);
    if (phase == 0 || phase == 2) {
        disp->drawDisc(cx - r + 1, cy - r/2, 1, U8G2_DRAW_ALL);
        disp->drawDisc(cx + r - 1, cy - r/2, 1, U8G2_DRAW_ALL);
    }
    if (phase == 1 || phase == 2) {
        disp->drawPixel(cx - r - 1, cy - r/2 - 1);
        disp->drawPixel(cx + r + 1, cy - r/2 - 1);
    }
    disp->setDrawColor(0);
}
```

## 一、核心Bug修复：眉毛动画完全未驱动舵机
这是你觉得「眉毛没活力」的**根本原因**：
`eye_expr_update()` 每帧都在计算 `brow_offset_l / brow_offset_r` 动画偏移，但这个偏移量只存在于变量中，**从未同步到物理舵机**。只有摇杆移动、切换表情的瞬间会更新一次舵机角度，后续所有眉毛微动、震颤、抽泣动画都没有实际输出。

### 修复步骤
#### 1. main.cpp 新增全局变量（保存摇杆偏移量）
在 `static uint32_t g_last_frame_ms = 0;` 下方添加：
```cpp
/* 摇杆带来的眉毛相对偏移（每帧与动画偏移叠加） */
static int8_t g_joy_brow_offset_l = 0;
static int8_t g_joy_brow_offset_r = 0;
```

#### 2. main.cpp 修改摇杆事件处理（替换原 EVT_JOYSTICK_MOVE 分支）
```cpp
case EVT_JOYSTICK_MOVE: {
    /* 1. 更新视线目标 */
    eye_set_look(&g_eyeCfg, msg->value_x, msg->value_y);

    /* 2. 保存摇杆眉毛偏移（不再直接设舵机，改为每帧合成） */
    int8_t brow_offset = msg->value_y / 8;
    g_joy_brow_offset_l = -brow_offset;
    g_joy_brow_offset_r =  brow_offset;
    break;
}
```

#### 3. main.cpp loop() 新增每帧舵机同步
在 `eye_expr_update(&g_eyeCfg, now);` 之后、渲染之前添加：
```cpp

    int8_t base_l = EXPRESSIONS[g_eyeCfg.active_expr].brow_left;
    int8_t base_r = EXPRESSIONS[g_eyeCfg.active_expr].brow_right;
    servo_set_target(
        base_l + g_joy_brow_offset_l + g_eyeCfg.brow_offset_l,

    servo_set_target(
```

# 我建议增加三个新的动画层（这是最值得做的）

这三个功能，我认为比继续调参数更能提升整体效果。

## ① Attention Layer（注意力层）

所有表情都应偶尔：

- 轻微转动视线。
- 停顿。
- 再回来。
- 配合眉毛极小幅跟随。

这样机器人会表现出“正在观察环境”，而不是始终盯着正前方。

------

## ② Secondary Motion（二级运动）

任何快速动作结束后，不要立即停止，而是加入惯性回弹：

- 开心时眉毛上扬后轻微回弹。
- 惊讶时眼睛张大后略微收回。
- 生气时眉毛下压后产生细微余震。

这种二级运动能显著增强动作的真实感。

------

## ③ Random Idle（随机待机）

即使保持同一个表情，也不要完全静止。可以随机触发一些很小的动作，例如：

- 瞳孔轻微缩放（±3%）。
- 单侧眉毛轻轻挑一下。
- 眼皮轻微收缩一下。
- 视线缓慢偏移 1～2 像素再回来。

这些动作不需要频繁出现，但能明显减少“机械循环感”。

------

## 我的总体评价

你现在的 **v7/v8** 已经把**渲染能力**做得不错了，问题已经不在于还能不能把眉毛再抖一点、眼泪再大一点，而是在于**缺少“行为学（Behavior）”层**。

如果继续只是调 `lid_top`、`brow_amp`、`pupil_scale` 等参数，提升会越来越有限；相反，引入**基础姿态 + 微动作 + 随机动作 + 二级运动 + 情绪节奏**的分层动画架构，会让整个机器人从“会做表情”提升到“像有情绪和生命力”。这也是目前许多高质量陪伴机器人和动画角色采用的设计思路。
