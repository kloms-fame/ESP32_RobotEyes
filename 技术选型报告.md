## 1. 项目概述

### 1.1 产品定义

**桌面陪伴机器人（Desktop Companion Robot）**，核心为「双OLED眼睛 + 舵机眉毛」的拟人化表情交互系统。

### 1.2 核心设计原则

| 原则 | 描述 |
|------|------|
| 双屏独立渲染 | 两块OLED分别对应左眼、右眼，支持异步动作和联动播放 |
| 视线控制分层 | 摇杆控制「Look Target」，经过平滑插值映射为眼球位置 |
| 舵机即眉毛 | 2个SG92R舵机控制左右眉，支持微动作和平滑过渡 |
| 事件驱动架构 | 所有输入统一封装为事件推入队列，与业务解耦 |
| 表情结构化 | 标准表情结构体，包含位图指针、动画参数、眉毛角度 |
| 生命感微动作 | 随机眨眼、视线漂移、眉毛微动等Idle动画 |
| 可扩展架构 | 预留传感器、灯光、语音等扩展接口 |

### 1.3 硬件规格

| 类别 | 型号 | 参数与接口 | 备注 |
|------|------|------------|------|
| 主控 | ESP32-C3 | RISC-V单核、400KB SRAM、4MB Flash | Arduino框架优先 |
| 左眼OLED | SSD1306 0.96寸 | 128×64 单色、四针I2C接口 | 独立渲染 |
| 右眼OLED | GME12864 | 128×64 单色、SPI接口 | 独立渲染，异接口 |
| 执行器 | SG92R舵机 ×2 | 50Hz PWM控制、0-180° | 左右眉毛 |
| 输入1 | LY-ADCKkeyboard-01 | ADC分压式8键键盘 | 短按/长按/双击 |
| 输入2 | XY模拟摇杆 | 2路ADC + 1路按键 | 注视目标控制 |
| 预留 | WiFi | ESP32-C3内置 | Web控制台 |

#### 2.1.1 目录结构与设计逻辑

```
u8g2/
├── csrc/                    # C语言核心实现
│   ├── u8g2.h               # 核心结构体定义
│   ├── u8g2_buffer.c        # 帧缓冲管理
│   ├── u8g2_bitmap.c        # 位图绘制
│   ├── u8g2_draw.c          # 基本绘图API
│   ├── u8x8_d_ssd1306.c     # SSD1306驱动
│   └── ...                  # 其他控制器驱动
├── cppsrc/                  # C++ Arduino封装
│   ├── U8g2lib.h            # 核心类定义
│   └── U8g2lib.cpp          # 类实现
├── sys/                     # 平台适配层
│   ├── arduino/             # Arduino平台
│   ├── esp32/               # ESP32平台
│   └── ...
└── tools/                   # 工具链
```

**设计范式**: 分层架构，C语言核心 + C++封装，适配器模式支持多种控制器和通信接口。

#### 2.1.2 软件整体架构

```
┌──────────────┐
│  U8G2 C++    │  Arduino面向对象封装
│   封装层     │
├──────────────┤
│   u8g2核心   │  C语言渲染引擎、帧缓冲管理
├──────────────┤
│   u8x8底层   │  显示控制器驱动、通信接口
├──────────────┤
│  硬件平台    │  ESP32/AVR/STM32等
└──────────────┘
```

#### 2.1.3 模块划分与职责

| 模块 | 职责 | 输入 | 输出 |
|------|------|------|------|
| u8g2_buffer | 帧缓冲管理 | 绘制命令 | 缓冲数据 |
| u8g2_draw | 图形绘制 | 坐标、尺寸 | 像素数据 |
| u8g2_bitmap | 位图渲染 | 位图指针、位置 | 显示输出 |
| u8x8_d_xxx | 控制器驱动 | 命令、数据 | 硬件信号 |
| u8x8_byte | 通信接口 | 字节数据 | I2C/SPI信号 |

#### 2.1.4 核心数据结构

```c
typedef struct u8g2_struct {
  uint8_t *buf;              // 帧缓冲指针
  u8g2_uint_t width;         // 显示宽度
  u8g2_uint_t height;        // 显示高度
  u8g2_uint_t tile_width;    // 瓦片宽度
  u8g2_uint_t tile_height;   // 瓦片高度
  u8x8_t u8x8;               // 底层驱动实例
  // ... 其他渲染状态
} u8g2_t;
```

#### 2.1.5 动画/渲染系统设计

- **双缓冲机制**: 离屏缓冲 + 一次性刷新
- **分页渲染**: `firstPage()` / `nextPage()` 循环分页输出
- **完整缓冲模式**: 适合128×64小屏，1KB RAM占用

#### 2.1.6 性能与优化

| 指标 | 数据 |
|------|------|
| 内存占用 | 1KB (128×64 full buffer) |
| 刷新帧率 | 30+ FPS |
| CPU占用 | 低（渲染操作在RAM完成） |

| `csrc/u8x8_d_ssd1306.c` | SSD1306驱动，支持I2C/SPI |

## 5. 绝对禁止重复造轮子清单

### 5.1 OLED驱动

**禁止自研**: U8g2已支持数百种控制器，包含SSD1306(I2C)和GME12864(SPI)，经过千万级设备验证。

**方案**: 使用U8g2库，实例化两个独立对象分别控制左眼(I2C)和右眼(SPI)。

### 5.2 舵机PWM控制

**禁止自研**: ESP32Servo基于LEDC硬件定时器，1μs精度，50Hz标准舵机信号。

**方案**: 使用ESP32Servo库，每个舵机独立attach，通过writeMicroseconds精确控制。

### 5.3 HTTP/WebSocket服务器

**禁止自研**: ESPAsyncWebServer异步非阻塞，支持WebSocket实时通信，社区成熟。

**方案**: ESPAsyncWebServer作为基础服务，ESPUI作为UI控件框架。

### 5.4 事件队列系统

**禁止自研**: Pixel-Box-ESP32的EventBus已实现线程安全的事件推送/消费模式。

**方案**: 直接引入EventBus.h，所有输入统一封装为EventMsg推入队列。

### 5.5 位图渲染API

**禁止自研**: U8g2提供完整的drawBitmap、drawXBM、drawXBMP等API。

**方案**: 使用U8g2的位图绘制函数。

### 5.6 FreeRTOS内核

**禁止自研**: Arduino ESP32框架已内置FreeRTOS，无需额外引入。

**方案**: 直接使用`xTaskCreate()`、`xQueueCreate()`等API。

### 6.4 模块通信机制

```
┌──────────────┐      EventBus      ┌──────────────┐
│ InputTask    │ ────────push──────► │ RenderTask   │
│ (ADC+摇杆)   │                    │ (U8g2渲染)   │
└──────────────┘ ◄───────push─────── └──────────────┘
      │                                      │
      │                                      │
      ▼                                      ▼
┌──────────────┐      EventBus      ┌──────────────┐
│ ServoTask    │ ────────push──────► │ BehaviorTask │
│ (眉毛控制)   │                    │ (情绪引擎)   │
└──────────────┘ ◄───────push─────── └──────────────┘
      │                                      │
      │                                      │
      └──────────────────────────────────────┘
                Shared Data (表情状态)
```

**通信规则**:

1. **事件队列**: 所有输入事件通过`EventBus`推送，各任务独立消费
2. **共享数据**: 表情状态、视线目标等通过线程安全的共享结构体访问
3. **互斥锁**: 共享数据访问需使用`xSemaphoreTake/Give`保护
4. **无直接调用**: 模块间禁止直接函数调用，仅通过事件或共享数据通信

### 8.3 事件总线使用（Pixel-Box-ESP32 EventBus）

```cpp
#include "EventBus.h"

void InputTask(void* parameter) {
    EventMsg msg;
    
    while (1) {
        // 读取ADC键盘
        int key = readADCKeyboard();
        if (key != KEY_NONE) {
            msg.type = EVT_KEY_PRESS;
            msg.value = key;
            msg.timestamp = millis();
            Event_Push(msg);
        }
        
        // 读取摇杆
        int x = analogRead(JOYSTICK_X_PIN);
        int y = analogRead(JOYSTICK_Y_PIN);
        
        msg.type = EVT_JOYSTICK_X;
        msg.value = x;
        Event_Push(msg);
        
        msg.type = EVT_JOYSTICK_Y;
        msg.value = y;
        Event_Push(msg);
        
        vTaskDelay(pdMS_TO_TICKS(30));
    }
}

void RenderTask(void* parameter) {
    EventMsg msg;
    
    while (1) {
        if (Event_Pop(&msg)) {
            switch (msg.type) {
                case EVT_JOYSTICK_X:
                    handleJoystickX(msg.value);
                    break;
                case EVT_JOYSTICK_Y:
                    handleJoystickY(msg.value);
                    break;
                case EVT_KEY_PRESS:
                    handleKeyPress(msg.value);
                    break;
                // ... 其他事件
            }
        }
        
        // 渲染循环
        renderFrame();
        vTaskDelay(pdMS_TO_TICKS(16)); // ~60fps
    }
}
```

### 8.4 舵机控制（ESP32Servo）

```cpp
#include <ESP32Servo.h>

Servo leftBrow;
Servo rightBrow;

void initServos() {
    leftBrow.attach(LEFT_BROW_PIN);
    rightBrow.attach(RIGHT_BROW_PIN);
    
    // 设置初始角度（眉毛放平）
    leftBrow.write(90);
    rightBrow.write(90);
}

void setBrowAngle(int8_t leftAngle, int8_t rightAngle) {
    // 限制角度范围
    leftAngle = constrain(leftAngle, 0, 180);
    rightAngle = constrain(rightAngle, 0, 180);
    
    // 平滑写入（ESP32Servo自动处理PWM）
    leftBrow.write(leftAngle);
    rightBrow.write(rightAngle);
}

void animateBrow(int8_t targetLeft, int8_t targetRight, uint16_t durationMs) {
    int8_t startLeft = leftBrow.read();
    int8_t startRight = rightBrow.read();
    
    uint32_t startTime = millis();
    
    while (millis() - startTime < durationMs) {
        float progress = (float)(millis() - startTime) / durationMs;
        
        // 缓动插值
        progress = easeInOutQuad(progress);
        
        int8_t currentLeft = startLeft + (targetLeft - startLeft) * progress;
        int8_t currentRight = startRight + (targetRight - startRight) * progress;
        
        leftBrow.write(currentLeft);
        rightBrow.write(currentRight);
        
        delay(10);
    }
    
    leftBrow.write(targetLeft);
    rightBrow.write(targetRight);
}

float easeInOutQuad(float t) {
    return t < 0.5 ? 2 * t * t : 1 - pow(-2 * t + 2, 2) / 2;
}
```
